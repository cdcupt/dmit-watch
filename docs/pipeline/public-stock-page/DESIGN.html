3 · Screen layout spec

  • Sticky zone — header + stale banner wrapped in one position:sticky container so the freshness signal never scrolls away on the ~6-screen wall. Header: brand (VPS · STOCK WATCH, mono 800) + one-line public subtitle; right side holds exactly one lamp: the freshness pill. When the stale banner is shown it pins directly under the header; the board dims but the sticky zone stays crisp.
  • Stats strip — the panel's stat-hero pill (“N in stock”, green when N≥1, muted when 0) + “M waiting · T watched” + per-location breakdown. This is the page's first read.
  • Jump chips — one horizontal, scrollable row of anchor chips (flag + DC code) linking to #dc-* sections; scroll-margin-top clears the sticky zone.
  • Explainer note — replaces the panel's “Routing Profile” note: one sentence on what the board is, how often it reads, and that buy links go to the provider cart. No marketing copy.
  • “In stock now” FIRST — order swapped vs the panel (which lists waiting first for the operator). Hero grid (min 240px columns), cards carry a context line (flag · city · CPU) because they are lifted out of their DC group.
  • Waiting wall — identical structure to the panel: DC bar (flag card + provider badge + count) → generation card (EPYC chip) → plan card grid (min 170px columns). Provider badge is data-driven: DMIT · Premium for DMIT families, qq.pw · WHMCS for WHMCS provider families.
  • Footer — how-it-works (3 short lines), “not affiliated” disclaimer, GitHub repo link. The only external link besides buy links.

5 · Freshness model (summary)

  • One source of truth: the server's receivedAt (time it accepted the last push). The page computes age = now − receivedAt locally, re-rendering the label every 1 s and re-fetching GET /api/state every 30 s.
  • Thresholds: fresh ≤ 2 min (green) · aging 2–5 min (amber pill only) · stale > 5 min (banner + dim + pulse) — matching PRD G3's 5-minute contract, with the amber tier as an early honest signal (~2 push cycles).
  • Unlike the local panel, freshness is never derived from connection liveness — a healthy page connection with old data must still read as old (consultant finding: the panel's 5 s SSE-based stale logic must not be reused as-is).

7 · Accessibility

  • Contrast — light background, near-black ink (existing token pairs all ≥ 4.5:1 for body text): --ink on --bg, --ok-ink on --ok-soft, --warn-ink on --warn-soft, white on --ok for pills/buttons (large/bold text). The dimmed stale board is deliberately low-contrast — the readable truth moves to the banner, which stays at full contrast.
  • Live regions — freshness pill and stale banner carry role="status" aria-live="polite"; only the pill's state class changes are announced-worthy — the per-second label tick must NOT re-announce (update text via textContent on the same node; screen readers treat it as one region, and the banner is the loud signal).
  • Keyboard — tab order: jump chips → Buy links → footer links. Cards are not tabbable (unlike the panel's tabindex="0" cards) — the only interactive element per card is the Buy link, keeping the tab path across 37 cards short. All interactive elements use the shared :focus-visible outline (2px blue, offset 2px).
  • Reduced motion — prefers-reduced-motion: reduce disables: FLIP card promotion, hover lift, stale-pill pulse, smooth anchor scrolling. State changes remain fully legible through color + copy.
  • Semantics — header / nav aria-label="Jump to datacenter" / section aria-labelledby per zone / footer; DC groups are sections with aria-label; status pills are text (not color-only) — every state has words.
  • Touch — Buy buttons and jump chips ≥ 40px effective hit area; buy links open in a new tab with rel="noopener".

10 · Implementation notes for Frontend (interaction contract)

TopicSpec
Entry filesSeparate public page + script (e.g. server/public/index.html + board.js), reusing tokens.css (trimmed copy) and the pure helpers (esc, fmtAgo, card HTML builder). Never import alarm.js, api.js POST wrappers, or tab logic.
DataGET /api/state → { ...state, receivedAt }; poll every 30 s (light jitter); re-render only when receivedAt changes; FLIP animation on section moves as in the panel.
FreshnessAge label ticks every 1 s from receivedAt; thresholds 120 s (amber) / 300 s (banner+dim). Never keyed off fetch success or SSE.
Status mappingunknown → checking at render time; sections decided by in vs not-in only (as the panel does).
Conditional fieldsspecs: render slot only if the snapshot carries spec fields (absent today — cards simply omit the row). inSinceMs exists → “in stock · Xm”. “out since” only if a future outSinceMs is pushed — do NOT promise it in copy until then.
OrderingWall keeps watchlist/config order (snapshot order); IN section in snapshot encounter order.
Head/metaTitle “VPS Stock Watch — live DMIT Premium availability”; meta description; no noindex; 👁️ favicon; color-scheme: light.
VerificationPer house testing rules: Playwright screenshots at 320/390/768/1440 for fresh AND stale states; zero horizontal overflow at all widths; reduced-motion pass.
